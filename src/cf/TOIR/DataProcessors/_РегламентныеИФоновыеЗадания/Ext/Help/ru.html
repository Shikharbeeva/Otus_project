Обработка является аналогом обработки из БСП, но с дополнительным функционалом, который, в некоторых случаях, просто незаменим.

Основные преимущества:

  • Работает отдельно от БСП (но можно подключить и как дополнительную обработку, если в конфигурацию встроена БСП).
  • Работает на платформе 8.2.
  • Быстрая работа со списком фоновых заданий (в типовой обработке при большом количестве фоновых заданий работать с ними невозможно).
  • Индикация состояния использования регламентных задания (включены/выключены), а также возможность включения/отключения блокировки регламентных заданий из обработки.
  • Избранные задания и отбор по ним.
  • Группировка по виду задания (актуально, если по одному виду задания есть несколько экземпляров).
  • Отбор по состояниию задания (активные, включенные, с ошибками и т.д.).
  • Поиск фоновых заданий по регламентному заданию (или виду задания), а также поиск регламентного по фоновому.
  • Отмена выполняющегося регламентного задания из списка (в типовой можно отменять только фоновое задание).
  • Запуск регламентного задания непосредственно (для удобства отладки).
  • Запуск регламентного задания как регламентного (в типовой обработке задание запускается как фоновое).
  • Просмотр всех параметров регламентного задания.
